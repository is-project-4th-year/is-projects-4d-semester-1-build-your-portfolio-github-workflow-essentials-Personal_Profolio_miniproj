3. Learnings from the Git Crash Program

What I Thought I'd Learn vs What I Actually Learned

1. Concept: Branching
Expectation : I assumed branching was only needed when working with big teams.
Reality : Even solo devs benefit! I could test new ideas safely.
Impact : Helped me develop my "About Me" page without breaking the homepage.

2. Concept: Commit Messages
Expectation : Just type "updated files" and move on, right?
Reality : Clear messages = a clear mind and clean history.
Impact : I could track what broke and when easily!

3. Concept: Issues and Milestones
Expectation : Thought issues were just for open source projects.
Reality : Helped me plan tasks and break down the project.
Impact : My milestone list gave me a launch checklist!

4. Concept: Pull Requests
Expectation : Pull requests are just for code reviews in teams.
Reality : It turns out PRs are useful for personal workflows too.
Impact : Made merging easier and changes more manageable.
